# 审查检查清单

## 通用检查项（所有 category）

### 代码质量
- [ ] 命名清晰，符合语言惯例
- [ ] 单函数不超过 50 行
- [ ] 圈复杂度不超过 10（无超过 3 层嵌套）
- [ ] 无明显复制粘贴代码
- [ ] 无硬编码的配置值、URL、密钥
- [ ] 外部调用有错误处理
- [ ] 不吞异常（无空 catch/except 块）
- [ ] 资源正确释放（文件/连接/锁）

### 设计一致性
- [ ] 实现与 feature 描述一致
- [ ] 每条验收标准有对应实现
- [ ] 遵循项目已有架构模式
- [ ] 不破坏已有接口
- [ ] 依赖方向正确，无循环依赖

### 测试质量
- [ ] 覆盖正常路径
- [ ] 覆盖异常路径
- [ ] 覆盖边界条件
- [ ] 测试有意义（非自欺欺人）
- [ ] 测试独立运行
- [ ] 断言精确具体

---

## data_model 类额外检查

### 代码质量
- [ ] 字段类型正确
- [ ] 必填/可选标注正确
- [ ] 约束定义完整（唯一、外键、检查约束）
- [ ] 默认值合理
- [ ] 索引定义合理

### 设计一致性
- [ ] 数据模型与 feature 中的数据字典一致
- [ ] 字段命名与已有模型风格一致
- [ ] migration 可回滚

### 测试质量
- [ ] 测试 CRUD 操作
- [ ] 测试约束生效（唯一约束、外键约束）
- [ ] 测试字段校验（长度、格式、范围）
- [ ] 测试默认值

## api 类额外检查

### 代码质量
- [ ] 参数校验完整
- [ ] 错误响应格式统一
- [ ] HTTP 状态码使用正确
- [ ] 幂等性（如适用）
- [ ] 分页/过滤实现正确

### 设计一致性
- [ ] API 路径与 feature 中的接口定义一致
- [ ] 请求/响应结构与 feature 描述一致
- [ ] 认证/鉴权逻辑正确

### 测试质量
- [ ] 测试正常请求（200/201）
- [ ] 测试参数缺失（400）
- [ ] 测试资源不存在（404）
- [ ] 测试权限不足（403）
- [ ] 测试边界输入

## ui 类额外检查

### 代码质量
- [ ] 组件职责单一

- [ ] 无内存泄漏（useEffect 清理、事件监听移除）

### 设计一致性
- [ ] 交互与 feature 中的界面描述一致
- [ ] 与全局交互规范一致
- [ ] 响应式适配（如需要）

### 测试质量
- [ ] 测试渲染正确
- [ ] 测试交互操作（点击、输入、提交）
- [ ] 测试加载态、空态、错误态
- [ ] 测试表单校验
